Structuur

Voordat je een website helemaal afmaakt is het verstandig om een idee te hebben van hoe je zou willen dat je website eruit ziet. Voor elke app die je maakt, is het verstandig om een ontwerp te maken. Het kan simpelweg gemaakt worden in een tekenprogramma, of zelfs met de hand als je dat fijner vindt.

Hiernaast zie je een foto van een voorbeeld layout die je zou kunnen gebruiken, maar er zijn natuurlijk ook andere indelingen mogelijk. HTML heeft een aantal tags die gebruikt kunnen worden om de opmaak van jouw website goed te krijgen. De volgende tags zijn erg handig:

<main> De hoofd inhoud van jouw website pagina. Er mag maar 1 main gebruikt worden per pagina en deze mag niet gebruikt worden in de tags hieronder.
<header> De kop van de website, hierin kan bijvoorbeeld een logo.
<footer> De onderkant van de website, hierin staan vaak contact gegevens.
<aside> Een zijbalk, deze is bijvoorbeeld te gebruiken bij een menu aan de zijkant.
<nav> De menubalk van de website, hiermee navigeer je over de site.
<article> Gebruikt om een stuk onafhankelijke inhoud weer te geven, denk bijvoorbeeld aan een nieuws artikel.
<section> Gebruikt voor een stuk bij elkaar horende informatie op de website.
<details> Een stuk informatie die verstopt kan worden door de gebruiker, bijvoorbeeld voor een begrippen lijst.
<summary> De titel van het detail blokje.

Opdracht 1

Geef voor de onderstaande blokjes code aan of ze wel of niet gebruikt mogen worden. Zet de antwoorden in jouw index pagina in een gesorteerde lijst met daarboven een h1 kopje met de tekst "Tag structuur:"

optie 1: 


  <main>
  </main>
  <main>
  </main>

optie 2: 


  <main>
    <aside>
    </aside>
  </main>

optie 3: 


  <nav>
    <main>
    </main>
  </nav>

Opdracht 2

Teken in paint, met draw.io of Powerpoint een schets van hoe je wilt dat jouw website opgebouwd is. Sla de foto of screenshot op in jouw mapje en voeg deze met behulp van de img tag toe aan jouw index. Hierboven staan nog twee extra voorbeelden. Voor meer inspiratie kun je ook googlen naar website layout. Zorg dat er minstens een header, nav, main, footer en een aside aanwezig is.
